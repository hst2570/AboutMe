Study list

  • [ 1. laravel ]

    • - PHP framework
      - spring framework와 비슷하다.
      - ORM을 표현하기위해 엘로퀸트 기능이 있다.
      - 데이터베이스 형상관리 기능인 Migration이 있다.
      - 데이터시드를 통해 테스트 데이터를 만들수 있다.
      - ServiceProvider를 사용해 DI(Dependency injection : 의존성 주입)가 가능하다.
      - Middleware는 http request 요청에 대해 전처리 또는 후처리 기능을 제공한다.
      - 라라벨 Auth기능으로 인증과 권한 기능을 쉽게 제작할 수 있다.
      - 템플릿은 blade라는 기능을 제공한다. 사용하기 편하다.

    • - 참고 사이트

      • laravel ( https://laravel.com/docs/5.3 )

  • [ 2. Spring framework ]

    • - Java 기반의 프레임 워크.
      - DI, AOP를 제공하고 POJO를 사용해 어플리케이션 제작이 가능하다.
      - ORM을 사용하기위해선 외부 라이브러리인 JPA를 사용하면 된다.
      - 어노테이션을 사용해 빈 등록이나 설정을 제어할 수 있다.
      - 하지만 어노테이션이 환경에 종속적인 정보를 갖는다면 더이상 POJO가 아니다.
      - Maven이나 Gradle을 사용해 많은 자바 라이브러리 사용이 가능하다.

    • - 참고 사이트

      • Spring framework reference

      • JPA 레퍼런스 (번역)

      • 한국 스프링 사용자 모임

  • [ 3. RESTful API ]

    • - 네트워크 기반 아키텍쳐
      - API를 사용할 때 CRUD를 http 메소드를 사용해 조작
      - 기존의 "user/add/{?}" 방식이 아닌 Post메소드를 사용한 user/{?} 방법
      - 메소드별 기능 = Get : 조회기능, Post : 새로운 데이터 삽입, Put : 업데이트, Delete : 삭제
      - 컬렉션과 엘리먼트를 나눠 조작한다. 컬렉션은 테이블과 비슷한 의미. 엘리먼트는 각 각의 데이터

    • - 참고 사이트

      • spoqa Tech Blog
